# NppDB Task Guide

Use this as a click-by-click guide. Open it again with `Show Tutorial` or `Ctrl+F11`.

## First setup: connect a database
1. Open `Database Connect Manager` (`F10`).
2. Click `Add a new database`.
3. Choose `MS Access` or `PostgreSQL` and fill the connection fields.
4. Click `Connect to the selected database`.
5. Expand the connection and tables in the tree.

## Generate SQL query from description
1. Open `Database Connect Manager` (`F10`) and select the relevant table or tables.
2. Open `Prompt Library` (`Ctrl+F10`).
3. Select `Generate SQL query from description`.
4. Fill the required `Prompt Inputs`.
5. Click `Copy Prompt` or `Open AI Chat`.

## Generate prompt from selected table
1. Open `Database Connect Manager` (`F10`).
2. Expand the table once so NppDB loads its columns.
3. Select the table. Use `Ctrl + Click` to select multiple tables.
4. Open `Prompt Library` (`Ctrl+F10`) and select a table prompt, for example `Explain table in plain language`.
5. Click `Copy Prompt` or `Open AI Chat`.

## Run selected SQL
1. Select the SQL text you want to run.
2. Click `Execute SQL` or press `F9`.
3. Read the result tab in the NppDB result panel.
4. If the database tree changed, click `Refresh database tree` in the result panel or `Refresh the selected database connection` in `Database Connect Manager`.

## Run one SQL statement without selecting it
1. Put the text cursor inside the statement.
2. Click `Execute SQL` or press `F9`.
3. Read the result tab in the NppDB result panel.

## Run multiple SQL statements
1. Select the whole block of SQL statements.
2. Click `Execute SQL` or press `F9`.
3. Check each result/message tab in the NppDB result panel.
4. Use result-tab right-click actions such as `Close all result tabs` or `Close other result tabs` if the panel gets cluttered.

## Analyze SQL
1. Select the SQL to analyze, or leave nothing selected to analyze the whole document.
2. Click `Analyze SQL` or press `Shift+F9`.
3. Read the highlighted issues and analysis messages.
4. Click `Clear analysis` or press `Ctrl+Shift+F9` when finished.

## Create an analysis prompt from SQL issues
1. Write or select the SQL you want checked.
2. Use `Analyze and Create Prompt (all queries)` (`Ctrl+F9`) for the whole editor.
3. Use `Analyze and Create Prompt (query at text cursor)` (`Alt+F9`) for only the issue under the cursor.
4. In the `Analysis Prompt` window, click `Copy to Clipboard` or `Open AI Chat`.

## Read table data
1. Open `Database Connect Manager` (`F10`).
2. Right-click a table.
3. Click `Select all rows`.
4. For MS Access, use `Select top 100 rows` if you only need a sample.
5. For PostgreSQL, use `Select random 100 rows` if you only need a sample.

## Export/read table as JSON or CSV
1. Open `Database Connect Manager` (`F10`).
2. Right-click a table.
3. Click `Select all as JSON` or `Select all as CSV`.
4. NppDB opens the output in a new tab.
5. The output is also copied to the clipboard.

## Generate CREATE TABLE SQL from a table
1. Open `Database Connect Manager` (`F10`).
2. Right-click a table.
3. Click `Generate CREATE TABLE statement`.
4. NppDB opens the generated SQL in a new tab and copies it to the clipboard.

## Useful shortcuts
- `F10` - `Database Connect Manager`
- `F9` - `Execute SQL`
- `Shift+F9` - `Analyze SQL`
- `Ctrl+Shift+F9` - `Clear analysis`
- `Ctrl+F9` - `Analyze and Create Prompt (all queries)`
- `Alt+F9` - `Analyze and Create Prompt (query at text cursor)`
- `Ctrl+F10` - `Prompt Library`
- `Ctrl+F11` - `Show Tutorial`

## Common problems

### `Copy Prompt` is disabled
1. Select a prompt in `Prompt Library`.
2. Fill all required `Prompt Inputs`.
3. Select or expand the needed table if the prompt needs table metadata.

### Multiple statements did not run
1. Select the whole SQL block.
2. Click `Execute SQL` or press `F9`.

### Analysis messages are still visible
1. Click `Clear analysis`.
2. Or press `Ctrl+Shift+F9`.

### The tutorial tab is hard to read after SQL highlighting
1. In Notepad++, open `Language`.
2. Select `Markdown`.
