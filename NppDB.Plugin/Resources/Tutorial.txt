# NppDB Tutorial

## Part 1 - Quick tutorial

## What NppDB does
- Connect to MS Access or PostgreSQL inside Notepad++
- Run SQL from the editor and view results
- Analyze SQL (warnings/errors + highlights)
- Generate copy-paste AI prompts from analysis
- Use reusable prompts from Prompt Library

## Most useful shortcuts
- `Ctrl + F11` - Open this tutorial again
- `F10` - Database Connect Manager
- `F9` - Execute SQL
- `Shift + F9` - Analyze SQL
- `Ctrl + Shift + F9` - Clear analysis highlights/messages
- `Ctrl + F9` - Analyze + AI prompt (all queries)
- `Alt + F9` - Analyze + AI prompt (query at text cursor)
- `Ctrl + F10` - Prompt Library

You can also find toolbar buttons for:
- Executing SQL (green triangle)
- Analyzing SQL (blue magnifying glass)
- Opening Prompt Library (book stack)
- Opening DB Manager (database icon)

## First launch setup
0. Switch into a different tab (as connections are per-tab)
1. Open Database Connect Manager (`F10`)
2. Add connection
   - MS Access: choose `.accdb` / `.mdb`
   - PostgreSQL: enter host / port / database / user / password
3. Connect and expand nodes to browse schemas, tables, views, and columns
4. Important: try right-clicking tables and columns in DB Manager. Many useful actions are there. You can also Ctrl + Click to select multiple tables

## Fast workflow
1. Connect with `F10`
2. Browse tables in DB Manager
3. Expand the table you need to so metadata loads
4. Open Prompt Library with `Ctrl + F10`
5. Pick a prompt, fill any extra placeholders, then copy it
4. Write or paste SQL in the editor
5. Select the exact block you want to run
6. Press `F9` to execute
7. Press `Shift + F9` to analyze if needed

## Run SQL (`F9`)
- If text is selected: executes statements inside the selection
- If nothing is selected: executes the statement your caret is currently inside
- If you want to run multiple statements, select all of them first

Results appear in NppDB's result/messages panel at the bottom of Notepad++.

## Analyze SQL (`Shift + F9`)
- If text is selected: analyzes the selection
- If nothing is selected: analyzes the whole document
- Issues show as highlights as well as in the analysis output

Clear analysis with `Ctrl + Shift + F9`.

## Prompt Library
- Open via plugin menu or `Ctrl + F10`
- Pick a prompt, fill missing placeholders, then press copy
- Required fields are marked with `*`
- Tags help describe prompts and make them easier to find

## Part 2 - Detailed tutorial

## Settings / Preferences
There are three tabs for settings:

- `Behavior`, which control specific aspects of plugin behavior, such as:
  - Opening new tab on query append and other contextual actions
  - Enabling destructive `SELECT INTO` statements for MS Access
  - Setting scale of text in Database Manager
  - Picking the color theme

- `LLM`, which contains settings for how AI responses are formatted and what info they include. This includes:
  - Response language
  - Custom instructions for a model (appended at the end of every prompt)
  - Option to change the URL pointed to by the `Open AI Chat` button in Prompt Library
  
- `AI Prompt Template`, where you can modify the template for 'Analyze + AI Prompt' feature

## DB Manager right-click actions
Do not treat DB Manager as only a tree view.

Right-click is important on:
- Connections
- Schemas
- Tables
- Columns

Depending on context, the right-click menu can contain things like:
- Viewing rows
- Selecting random rows
- Table-level quick actions
- Connection actions

If you are not sure whether something has a menu, try right-clicking it.

## SQL execution behavior
When you press `F9`:
- If text is selected: NppDB runs the selected text
- If nothing is selected: NppDB runs the statement at the caret

Important!
- If you want to run multiple statements, select the whole block first

## SQL analysis behavior
When you press `Shift + F9`, the entire tab is analyzed.
If you wish to analyze only the issue at text cursor, use `Alt + F9`, or `Analyze and Create Prompt (query at text cursor)` option in the menu. 

Useful related shortcuts:
- `Ctrl + Shift + F9`: clear analysis highlights and messages
- `Ctrl + F9`: create one AI prompt from all detected issues

Generated prompts are copied to the clipboard automatically.

## Prompt Library - what it is for
Prompt Library contains reusable prompts such as:
- Explaining a query
- Optimizing a query
- Debugging wrong results
- Converting SQL dialects
- Generating documentation

Prompt Library prompts can use:
- selected SQL from the editor
- table metadata from DB Manager
- additional user-filled placeholders

## Prompt Library - basic use
1. Open Prompt Library with `Ctrl + F10`
2. Select a prompt from the list
3. Read the description and tags
4. Fill required fields marked with `*`
5. Press the copy button
6. Paste the result into your preferred LLM

Important behavior:
- If no SQL is selected, prompts using selected SQL use the whole current editor text
- Some fields may be auto-filled and read-only
- `{{table_name}}` is filled from the selected table in DB Manager
- `{{table}}` is filled from table metadata; expand the table once in DB Manager first
- Copy stays disabled until required fields are filled

## Prompt Library - tags and search
Tags are keywords attached to prompts.

They are useful because:
- They describe what a prompt is for
- They make prompts easier to find
- They are shown under prompts in the list

Search checks:
- Prompt name
- Description
- Tags

You can search with:
- Normal words, for example `performance` or `documentation`
- Tag search, for example `#performance` or `tag:analytics`

Good tag examples:
- `performance, optimization, indexing`
- `schema, explain, onboarding`
- `api, rest, specification`

## Prompt Library - editing
Prompt Library also supports editing.

You can:
- Add a new prompt
- Duplicate an existing prompt
- Delete a prompt
- Edit prompt name
- Edit prompt description
- Edit prompt tags
- Edit prompt text/template

Notes:
- Turn on editing mode to modify prompts
- Changes are auto-saved
- Placeholders use `{{placeholder_name}}` syntax
- Useful built-in placeholders:
  - `{{selected_sql}}` - selected SQL, or the whole current file if nothing is selected
  - `{{dialect}}` - current database dialect
  - `{{table_name}}` - selected table name from DB Manager, or multiple selected table names separated by commas
  - `{{table}}` - selected table metadata, or multiple selected table definitions
- Tags should be short keywords separated by commas

## Common problems
- "No database connection attached"
  - Open `F10`, connect to a database, then retry `F9`

- Prompt says `Please expand table in DB Manager`
  - Expand the table node once so column metadata loads, then retry

- Copy button in Prompt Library looks unavailable
  - Usually no prompt is selected, an editable placeholder like `{{user_request}}` is empty, or required auto-filled context is missing

- Several statements did not all run
  - Select the whole block first, then press `F9`
  
- Tab colors are all messed up! I can't read tutorial like this!
  - Connecting to database or performing analysis changes tab language to SQL. Change it back in Notepad++ in "Language -> Markdown"
